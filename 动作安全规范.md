# 动作安全规范

## 1. 目标

这份规范的目标不是“绝对保证硬件永不受损”，而是把现有官方动作库里已经验证过的运动包络提取出来，用它约束自定义动作和动作拼接，尽量把风险压到官方样本范围内。

当前项目已经做到两件事：

1. 能把官方 `.rob` 动作解析成明文 `ACT-40` 数据。
2. 能用官方动作库自动学习安全包络，并审计新动作是否越界。

## 2. 当前采用的安全基线

安全基线来自 `动作` 目录中的官方明文动作样本。

当前统计结果：

1. 参考动作文件数：82
2. 参考动作帧数：2302
3. 单帧时长范围：40 到 1800
4. 单帧时长 95 分位：800
5. 相邻帧总姿态跳变量 `L1` 的 95 分位：1814
6. 相邻帧总姿态跳变量 `L1` 的最大值：3694

这里的“总姿态跳变量”是前 16 个有效关节第 1 字段的逐关节绝对差之和。

## 3. 当前采用的硬约束

新的明文动作文件必须满足下面这些硬约束：

1. 每帧时长必须落在官方范围内。
2. 前 16 个有效槽位的三个字段，都必须落在官方样本观测到的范围内。
3. 相邻帧之间，每个关节第 1 字段的跳变不能超过官方最大值。
4. 相邻帧的总姿态跳变量不能超过官方最大值。

如果超过这些约束，`rob_safety.py` 会直接报错，而不是把文件当作“可能可用”。

## 4. 当前采用的软约束

除了硬约束，还保留一层更保守的提醒规则：

1. 如果动作段拼接处的总姿态跳变量超过官方 95 分位，就标记为 `warn`。
2. 如果没有超过 95 分位，但落在官方常见区间内，则标记为 `ok`。

这层规则不是为了阻止所有动作，而是提醒哪些拼接已经接近官方动作库的高强度边界。

## 5. 为什么只重点看前 16 个槽位

因为现有格式已经确认：

1. 每帧共有 40 个槽位。
2. 其中只有前 16 个槽位是真实有效动作槽位。
3. 后 24 个槽位是固定 filler `(0x5555, 0, 0)`。

所以真正会改变机器人姿态的，是前 16 个槽位。

## 6. 为什么三个字段都要检查

目前第 2、第 3 字段的精确语义还没有完全源码级还原，但它们显然属于官方动作的一部分。

因此当前安全策略是：

1. 第 1 字段当作主要姿态目标值，严格做范围和跳变检查。
2. 第 2、第 3 字段虽然语义未完全反编译，但仍然必须落在官方样本包络内。
3. 在没有完全理解轮函数之前，不手工造这两个字段，只复用官方动作里已有的值。

这是当前最稳妥的工程做法。

## 7. 对编舞的实际约束

如果要继续拼新舞蹈，建议遵守下面几条：

1. 优先复用官方动作帧，不直接手写三字段数值。
2. 新动作段之间必须先过 `rob_safety.py` 审计。
3. 优先选开始和结束都接近立正位的官方动作做拼接锚点。
4. 对起始姿态偏离较大的动作，最好放在前一个动作已经回正之后再接入。
5. 不要为了追求节奏强行压缩官方动作帧时长到 40 以下。

## 8. 当前自制舞蹈结论

当前自制舞蹈文件：

- `动作/159号自制舞蹈.rob`

已经过审计，结果如下：

1. 总帧数：49
2. `violations=0`
3. 最大相邻帧总姿态跳变量：1916

当前动作段拼接结果：

1. `intro-stand -> wave`：`ok`，`max=75`，`l1=167`
2. `wave -> twist-1`：`ok`，`max=338`，`l1=1703`
3. `twist-1 -> twist-2`：`ok`，`max=338`，`l1=1702`
4. `twist-2 -> march`：`ok`，`max=68`，`l1=219`
5. `march -> pose`：`ok`，`max=0`，`l1=0`
6. `pose -> bow`：`ok`，`max=197`，`l1=955`
7. `bow -> outro-stand`：`ok`，`max=8`，`l1=15`

也就是说，这支舞蹈虽然在局部拼接上接近官方高强度区间，但仍然没有越过当前学习出的硬安全边界。

## 9. 现有工具

当前已经可用的工具有：

1. `rob_reverse.py`：解析 `.rob` 文件结构。
2. `rob_decrypt_with_upper.ps1`：把 `EYPT` 动作解成明文 `ACT-40`。
3. `rob_safety.py`：审计动作文件是否落在官方安全包络内。
4. `rob_compose.py`：生成新舞蹈，并在写出前自动执行安全检查。

## 10. 仍然不能声称绝对安全的部分

下面这些因素，代码层面无法完全保证：

1. 机器人当前机械装配是否偏心或卡滞。
2. 舵机老化、温升、负载变化。
3. 地面摩擦和支撑条件变化。
4. 电池电压过低造成的姿态失稳。
5. 第 2、第 3 字段的物理意义仍未完全源码级还原。

所以当前结论应该表述为：

“新动作已经被限制在官方动作样本的已知安全包络内，并通过了自动审计；但硬件安全仍然需要结合实际装配、供电和地面条件验证。”
