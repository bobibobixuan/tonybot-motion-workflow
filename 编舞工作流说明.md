# 编舞工作流说明

## 1. 目标

这条工作流是把“用户一句话需求”落到机器人动作文件的一条标准链路：

1. 先做网页检索和动作总结。
2. 把总结结果写入结构化 JSON。
3. 用 JSON 驱动编译器生成 `.rob`。
4. 自动执行安全审查。
5. 额外生成一份 HTML 时间线可视化。

## 2. 工作流结构

标准产物有三类：

1. 编舞规格 JSON：描述舞蹈意图、网页研究结果和动作段。
2. 编译结果 `.rob`：最终给机器人使用的动作文件。
3. 报告与可视化：包含安全结论、拼接边界和动作时间线。

## 3. 代理执行顺序

如果后面你直接说“我要一个某某风格的舞蹈”，推荐按下面顺序执行：

1. 先上网查这个舞蹈的关键动作、节奏、开场和收尾习惯。
2. 把结论写进 `编舞/<舞蹈名>.json` 的 `research` 字段。
3. 把实际选用的官方动作段写进 `segments`。
4. 运行 `python dance_workflow.py build 编舞/<舞蹈名>.json`。
5. 查看生成的 HTML 时间线和安全报告。

## 4. 命令

初始化一个空模板：

```powershell
python dance_workflow.py init 编舞/示例舞蹈.json --name 示例舞蹈 --prompt "做一段节奏明显、适合展示的机器人舞蹈"
```

根据 JSON 构建 `.rob`、报告和时间线：

```powershell
python dance_workflow.py build 编舞/示例舞蹈.json
```

如果只想单独用编译器，也可以直接走：

```powershell
python rob_compose.py 编舞/示例舞蹈.json
```

## 5. JSON 关键字段

### `prompt`

用户原始需求，例如：

- “做一段开心、适合比赛展示的街舞”

### `research`

这里保存网页研究结果：

1. `query`：搜索关键词。
2. `summary`：提炼出的动作要点。
3. `references`：参考网页链接。

### `segments`

这里是最终落到机器人动作库的动作段，支持字段：

1. `source`：动作库中的 `.rob` 文件名或路径。
2. `label`：时间线显示名称。
3. `repeat`：重复次数。
4. `frame_range`：可选的帧截取范围，格式为 `[start, end)`。
5. `notes`：为什么选这个动作段。

## 6. 输出文件

构建后通常会得到：

1. `动作/<舞蹈名>.rob`
2. `编舞/<舞蹈名>.report.json`
3. `编舞/<舞蹈名>.timeline.html`

## 7. 当前约束

这条工作流现在已经能完成“网页总结 -> JSON -> `.rob` -> 安全审查”，但动作段选择仍然依赖现有官方动作库。

也就是说：

1. 它不会凭空生成全新关节轨迹。
2. 它会优先从官方动作库里选段、裁剪、重复、拼接。
3. 安全边界仍以官方动作样本包络为准。

这也是当前最稳妥的方案。
